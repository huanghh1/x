# 币安二级监控系统 - 2026-06-13 实现总结

> 本节为当前真实状态；下方旧 Phase 记录仅保留作历史参考。

## 当前完成状态

- 资金费率、OI、关注池、热度、均线和触发历史均使用真实数据库字段。
- 同代币四周期聚合为一行，展开后可切换周期图表。
- 热度排行严格分链、分页，并排除市值前 10、稳定币和代币化股票。
- 推特 token 池会识别额度不足/频率限制，失败不再显示为数值 0。
- 资金费率、热度、多周期、OI 按统一优先级关联；普通一级警报不单独发 TG。
- TG 正文包含每个代币的复制代码、推特和 Binance Square 搜索链接。
- API、crawler、realtime、scheduler 已拆为四个进程。
- MySQL 增加查询索引、批量写入、唯一键去重、`SKIP LOCKED` 和定期清理。
- 关注池支持下一次解锁日期缓存；未配置数据源时明确显示状态。
- 图表支持横向缩放/平移、右侧价格轴纵向缩放、价格轴拖动和 Shift 纵向平移。
- 测试、JavaScript 语法检查、数据库迁移与浏览器逐页验收已纳入交付检查。

## 待完成的改进 (Phase 5-8)

### Phase 5: 信号匹配与排序规则 🟠
需要实现等级排序系统：
```
优先级从高到低：
1. 资金费率 + IO + 热度 + 多周期 (🔥🔥🔥🔥)
2. 资金费率 + 热度 + 多周期 (🔥🔥🔥)
3. IO + 热度 + 多周期 (🔥🔥⭐)
... (12个等级)
```

TG推送规则：
- 级别1-12发送TG
- LEVEL1警报及以下不发送（需求19）

### Phase 6: 问题排查与优化 🟠
- **热度排行推特为0**：✅ 已识别空额度假成功，启用 token 池轮换、失败冷却和后台刷新
- **分链不生效**：✅ 已按链校验返回数据、隔离失败缓存，并取消前端过期请求
- **响应慢**：✅ Twitter 和通知移出响应主路径，增加共享分链缓存
- **TG连接超时**：增加超时时间、添加重试逻辑
- **TG按钮响应慢**：异步处理、消息队列
- **币安广场搜索**：跳转到搜索结果页面
- **市值前10过滤**：✅ CoinGecko 动态前 10；同时过滤稳定币和代币化股票

### Phase 7: TG消息集成与优化 🟠
需要将以下操作从按钮集成到消息中：

- Twitter搜索链接

消息格式改为：
